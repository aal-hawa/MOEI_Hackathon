## 1. MINISTRY OVERVIEW

The Ministry of Energy and Infrastructure (MOEI) was formed by merging the former Ministry of Energy and Ministry of Infrastructure. It is headed by Minister Suhail Al Mazrouei. The Undersecretary for Energy and Petroleum Affairs is HE Eng. Sharif Al Olama.

**Vision (Arabic):** الريادة العالمية المستدامة في الطاقة والمياه والبنية التحتية والإسكان والنقل  
**Vision (English):** Global sustainable leadership in energy, water, infrastructure, housing, and transportation

**Mission (Arabic):** تنظيم وتطوير وتعزيز القدرة التنافسية في الطاقة والتعدين  
**Mission (English):** Organize, develop, and enhance competitiveness in energy, mining, infrastructure, housing, and transport

## 3. COMPLAINT PROCESS & FIELDS

### 3.1 Complaint Submission Flow

1. **Access:** Via MOEI website (moei.gov.ae) → Contact Us → Submit a Complaint
2. **Authentication:** UAEPASS login required
3. **Form Completion:** Fill complaint details
4. **Submission:** Receive complaint reference number
5. **Tracking:** Track via reference number on MOEI website
6. **Resolution:** MOEI Call Center (800-6634) follows up

### 3.2 Complaint Categories
- Electricity service complaints
- Water service complaints
- Infrastructure complaints
- Transportation complaints
- Petroleum service complaints
- Housing program complaints
- General service complaints

| Field | Arabic | Required | Type |
|-------|--------|----------|------|
| Complainant Name | اسم مقدم الشكوى | YES | Text (auto-filled from UAEPASS) |

| Service Reference Number | رقم مرجع الخدمة | NO | Text |
- Attachments | المرفقات | NO | File upload |

### 3.4 Electricity/Water Complaint Specifics
For utility complaints (electricity/water), the process typically involves:
- **First step:** Contact the specific utility provider (DEWA, ADDC, SEWA, FEWA, etc.)
- **Escalation:** If unresolved, escalate to MOEI
- **Required info:** Account number, meter reading, bill details, issue description
- **Note:** MOEI is the federal regulator; local utilities handle direct service

## 5. UAEPASS - NATIONAL DIGITAL IDENTITY

### 5.1 What is UAEPASS?
- **Official Name:** UAE PASS (البوابة الرقمية الموحدة)
- **Launched:** Gitex Exhibition 2018
- **Type:** First national digital identity solution for UAE
- **Website:** https://uaepass.ae
- **Self-care Portal:** https://selfcare.uaepass.ae
- **Help Desk:** 600 561 111
- **Collaborators:** Digital Dubai, TDRA (Telecommunications and Digital Government Regulatory Authority), Department of Government Enablement

### 5.2 UAEPASS ID / Login Identifiers
Users can log in using:
- Mobile number
- Email address
- Emirates ID number

### 5.3 Emirates ID Number Format (Linked to UAEPASS)

**Format:** `784-NNNN-NNNNNNN-N` (15 digits total with optional hyphens)

**Structure (4 parts):**
| Part | Digits | Meaning |
|------|--------|---------|
| Part 1 | 3 digits | **784** - ISO 3166-1 numeric code for UAE |
| Part 2 | 4 digits | Year of birth (generally, but not always - randomly generated) |
| Part 3 | 7 digits | Random unique identifier |
| Part 4 | 1 digit | Check digit (Luhn algorithm) |

**Example:** `784-1990-1234567-1`

**Regex Pattern:** `\b784[0-9]{4}[0-9]{7}[0-9]{1}\b`

**Validation:** Must pass Luhn check algorithm

**Note:** The 15-digit number is permanent and randomly generated. Even if some digits suggest birth year, the number does not change.

### 5.5 UAEPASS Features
1. **Face Recognition (Facial ID):** Register with biometric face recognition, creating verified account in <5 minutes
2. **Digital Vault:** Store and share digital documents via blockchain-powered vault
3. **Digital Signature:** Sign documents digitally (legally binding)
4. **Single Sign-On (SSO):** One login for all government and private sector services
5. **Profile Management:** Update profile information and address records
6. **Document Verification:** Verify signed documents instantly

### 5.6 UAEPASS Integration with Government Services

**OAuth2 Flow:**
- **Authorization URL:** `https://id.uaepass.ae/idshub/authorize`
- **Scope:** `urn:uae:digitalid:profile:general`
- **ACR Values:** `urn:safelayer:tws:policies:authentication:level:low` (and higher)
- **Client ID Format:** e.g., `tra_gov_web_prod` (for u.ae portal)
- **Redirect URI:** Service-specific callback URL

**Integration Process:**
1. Service provider registers with UAEDID
2. Receives client_id and credentials
3. Implements OAuth2 authorization flow
4. Receives user profile data upon consent
5. Returns normalized identity attributes

**Integration Challenges (from research):**
- Direct API integration requires UAEDID registration and developer credentials
- UAE PASS returns raw identity attributes requiring normalization
- Months-long integration process for most institutions
- No standardized data format across responses

**CBUAE Alignment:** UAE PASS is endorsed by the Central Bank of UAE as part of digital financial services framework, increasingly required for compliant digital onboarding under UAE AML/CFT requirements.

## 10. KEY INTEGRATION PATTERNS FOR SERVICE RULES SYSTEM

### 10.1 Service Submission Pattern
```
1. User authenticates via UAEPASS
2. UAEPASS returns: Full Name, Emirates ID, DOB, Nationality, Gender, Mobile, Email
3. User selects service category
4. System pre-fills identity fields from UAEPASS data
5. User fills service-specific mandatory fields
6. User uploads required documents
7. System validates all mandatory fields
8. System generates reference/tracking number
9. User receives confirmation (SMS + email)
10. Service enters processing workflow
```

### 10.2 Emirates ID Validation Rules
```typescript
// Emirates ID validation
const EID_REGEX = /^784\d{4}\d{7}\d{1}$/;
const EID_FORMATTED = /^784-\d{4}-\d{7}-\d{1}$/;

// Luhn check digit validation required
function validateEmiratesID(eid: string): boolean {
  const digits = eid.replace(/-/g, '');
  if (digits.length !== 15) return false;
  if (!digits.startsWith('784')) return false;
  return luhnCheck(digits);
}
```

### 10.3 UAE Mobile Number Format
```
Format: +971-XX-XXX-XXXX or 0XX-XXX-XXXX
Mobile prefixes: 50, 52, 54, 55, 56, 58 (Etisalat, du, Virgin Mobile)
```
